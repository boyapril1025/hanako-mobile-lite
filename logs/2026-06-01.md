### 已确认的关键发现

1. **`evaluateJavascript` IPC 路由问题**：`onPageFinished` 时调用 `evaluateJavascript` 会跑在 `about:blank` 空文档上，不是渲染页面。诊断脚本 `document.querySelectorAll('*')` 返回 0 个元素。
2. **`loadUrl("javascript:...")` 可用**：改用 `loadUrl` 后诊断正常，能看到 420 个 DOM 元素。
3. **`loadUrl` 的 URL 截断**：URL 不允许原始换行符 `\n`。Kotlin 中 `\\n` 产生两字符 `\n`（安全），`\n` 产生真实 LF（危险，URL 被截断）。
4. **Bootstrap 轮询正常**：`injectBootstrapLoader()` 用 `loadUrl` 注入，轮询 `document.readyState` 和 `.mobile-desktop-root`，100ms 就绪。

### 未解决的两个问题

#### 问题 1：返回手势不回到聊天列表

**目标**：Android 返回手势 → 打开侧边栏（聊天列表）；侧边栏已打开 → 退出 App。

**当前方案**：
- `injectBackNavigation()` 用 `loadUrl` 注入 `window.__hanaHandleBack` 全局函数
- `setupBackPress()` 用两步法：`loadUrl` 执行 handler → 100ms 后 `evaluateJavascript` 读取结果
- sidebar 检测：`[class*="sidebar" i]:not(.collapsed)` + `offsetWidth > 0`
- 点击 `#tbToggleLeft` 打开侧边栏

**问题**：注入没有 SyntaxError，但返回手势不工作。可能原因：
- sidebar 的 `collapsed` class 逻辑可能和预期不同
- `tbToggleLeft` 点击后 React 状态没有更新
- 两步法的 100ms 延迟可能不够
- 需要进一步调试：在 `__hanaHandleBack` 中加 `console.log` 确认函数被调用

#### 问题 2：Enter 键发送而不是换行

**目标**：Enter 键在 ProseMirror 编辑器中插入换行，不发送消息。

**当前方案（Hanako 建议）**：
- capture 阶段拦截真实 Enter（`isTrusted=true`）
- `preventDefault + stopImmediatePropagation` 阻止 ProseMirror 和 React 处理
- 派发合成 KeyboardEvent 到 ProseMirror 元素（`isTrusted=false`）
- `!e.isTrusted` 守卫防止死循环
- ProseMirror 的 keymap 处理合成事件，走正规 Transaction

**问题**：注入没有 SyntaxError，但 Enter 键仍然发送。可能原因：
- ProseMirror/Tiptap 的 keymap 可能检查 `isTrusted` 并忽略合成事件
- React 外层的发送逻辑可能不依赖 keydown 事件（可能监听 input 或其他事件）
- 合成事件可能没有被 ProseMirror 的 `handleKeyDown` 正确接收
- 需要进一步调试：确认合成事件是否到达 ProseMirror

### 调试中踩过的坑

1. **`evaluateJavascript` vs `loadUrl`**：`evaluateJavascript` 在 `onPageFinished` 时跑在 about:blank。所有注入必须用 `loadUrl`。
2. **`loadUrl` 多行 JS**：URL parser 在 `\n` 处截断。所有 `loadUrl` 注入必须是单行。
3. **IIFE 括号匹配**：Kotlin 字符串拼接的 JS 容易漏括号。最终用无 IIFE 的 `if(!window.__xxx){...}` 语法避免问题。
4. **`execCommand` 在 ProseMirror 中无效**：ProseMirror 的 mutation observer 会回滚非 Transaction 产生的 DOM 变更。
5. **`className` 对 SVG 元素不安全**：SVG 元素的 `className` 是 `SVGAnimatedString` 不是 `string`。
6. **`isComposing` 守卫**：Android IME 的 Enter 可能在 composition 期间触发。但 `isComposing` 守卫导致 handler 跳过，事件到达外层触发发送。最终方案用合成事件解决。

### 接手建议

1. **优先确认合成 KeyboardEvent 是否被 ProseMirror 接受**：在 `__hanaHandleEnter` 中加 `console.log('dispatched')`，看是否输出。如果输出但 Enter 仍然发送，说明 ProseMirror 忽略了合成事件。
2. **考虑替代方案**：如果合成事件不行，试 `insertLineBreak`（不需要换行参数）或 `insertHTML('<br>')`。
3. **返回手势调试**：在 `__hanaHandleBack` 中加 `console.log`，按返回键时看 logcat 是否输出。如果没有输出，说明函数没被注入或没被调用。
4. **考虑 WebView 调试**：`WebView.setWebContentsDebuggingEnabled(true)` 后用 PC Chrome 的 `chrome://inspect` 直接操作 DOM，比盲调快得多。
5. **备份位置**：`backups/v0.1.5-pre-upgrade-20260601-143028/`，包含 APK、源码、Hanako 服务端 mobile UI 文件。

## 2026-06-01 晚间：Hanako 接手修复

### 返回手势修复（`__hanaHandleBack` 重写）

**根因**：阮秀方案用 `[class*="sidebar" i]:not(.collapsed)` 选择器检测侧边栏可见性，但诊断显示 sidebar class 始终含 `collapsed`，选择器永远不匹配。此外侧边栏可见时返回 `false` 导致 `setupBackPress` 走 `webView.goBack()` 而非退出 App。

**修复**：
1. 不依赖 class，改用几何检测：`offsetWidth > 50` + `getComputedStyle` 判断侧边栏是否真实可见。
2. 改成三态返回值：`'handled'`（已打开侧边栏，吃掉事件）、`'exit'`（侧边栏已可见，退出应用）、`'back'`（找不到侧边栏，fallback 到 WebView goBack）。
3. `setupBackPress` 改用 Kotlin `when` 分支处理三种状态，`'exit'` 直接禁用 callback 并退出。

### Enter 键修复（`__hanaEnterFix` 重写）

**根因**：
1. 监听在 `document` capture 阶段太晚——React/ProseMirror 可能在 `window` 层已处理 keydown。
2. 合成 `KeyboardEvent`（`isTrusted: false`）可能被 ProseMirror/Tiptap 的 keymap 忽略。
3. `insertText('\n')` 在 Kotlin 源文件中 `\n` 被编译为真实换行符，`loadUrl` 的 URL 遇到换行符会被截断（阮秀已验证的坑）。

**修复**：
1. 监听提升到 `window` capture 阶段（最早介入点）。
2. 新增 `getPMView()` 辅助函数，通过 `pmViewDesc` 链和 React fiber 两种方式获取 ProseMirror EditorView 实例。
3. 优先方案：`view.dispatch(view.state.tr.insertText('\\n'))` — 走 ProseMirror 正规 Transaction，不依赖合成事件。
4. 回退方案：直接 DOM 插入 `<br>` + 调整选区（当无法获取 ProseMirror view 时）。
5. `insertText` 参数改为 `'\\\\n'`（Kotlin 编译为 `'\\n'`，JS 解读为换行符），避免 loadUrl 截断。

## 2026-06-01 晚间 Part 2：真机反馈后二次修复（Hanako）

### 问题 1：JS SyntaxError `Unexpected token ')'`

**根因**：Enter fix 注入的 IIFE 缺少外层函数体闭合 `}`。括号平衡检测：花括号 +1（`(function(){` 有开无闭，结尾缺一个 `}`）。结尾 `)())` 中 `)` 关闭了 grouping paren，但 `{` 没关。

**修复**：结尾从 `}},true)())` → `}},true)})()`，补齐外层函数体闭合括号。

### 问题 2：返回手势仍切出 APP

**根因**：`setupBackPress` 中 `loadUrl(` 执行 handler + `postDelayed 100ms` + `evaluateJavascript` 读取结果，三步存在竞态。`loadUrl` 是异步的，100ms 后 `evaluateJavascript` 执行时 `__hanaBackResult` 可能尚未写入（读到 `null`），fallback 到 `"back"` → `webView.goBack()` → 页面回到空白。

**修复**：去掉两步法，直接 `evaluateJavascript("(window.__hanaHandleBack&&window.__hanaHandleBack())||'back'")`，消除竞态。页面已加载时 `evaluateJavascript` 能正确在主 frame 执行。
